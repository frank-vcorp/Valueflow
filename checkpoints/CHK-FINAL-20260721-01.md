## 🎨 Decisiones arquitectónicas clave

| Decisión | Justificación |
|----------|---------------|
| **Driver Firebird: `node-firebird-native-api`** | Soporte oficial Firebird 5.0, usa `fbclient.dll` incluida con SAE 10 |
| **API Key en `config.json`** (no `.env`) | Permite actualización via UI sin reinicio ni editar archivos |
| **`maskApiKey()` en logger** | NUNCA se loguea la API Key completa |
| **Crons independientes** (`0 2 * * *` inventario, `0 3 * * *` ventas) | Try/catch aislados por job, un fallo no afecta al otro |
| **UI local con logos Repaga + Approved Partner** | Refuerza identidad visual + valida relación comercial Siemens |
| **Acceso directo a Firebird** (no reportes SAE) | Más rápido, completo, sin intervención manual |
| **Solo campos básicos requeridos** | 5 inventario + 10 ventas; opcionales como toggles en UI |
| **Filtro de marca Siemens** | 15 líneas en `INVE01.LIN_PROD`; solo se reportan productos marca Siemens |
| **Reintentos con backoff exponencial** | 2s → 4s → 8s → 16s → 32s, máximo 60s, solo 5xx |
| **PM2 + pm2-windows-startup** | Auto-arranque como Servicio Windows |
| **Winston + daily-rotate** | Logs JSON estructurados, retención 90 días |
| **Pool de BD 1-5 conexiones** | Compartido entre jobs pero con capacidad para concurrentes |
| **Basic Auth en UI** | Protección simple y efectiva para entorno local |

## 📊 Observaciones importantes

### 1. Intermitencia del sandbox QUA de Siemens
- **Patrón confirmado:** el servidor QUA pasa de UP (HTTP 201) a DOWN (HTTP 502) sin causa aparente
- **Acción pendiente:** contactar al **Siemens Data Steward regional** para reportar
- **Mitigación:** middleware tiene reintentos robustos; go-live debe ser directo a PRD
- **Documentado en:** `MEMO_DIAGNOSTICO_ERROR_502_SIEMENS.md`

### 2. Limitación técnica en entorno de desarrollo
- `npm install` no completó 100% en Linux (Node 22, ruta con espacios bloquea node-gyp)
- **Esto NO es bug del código** — se resuelve en Windows con Node 20 LTS + `fbclient.dll` de SAE 10
- Workaround usado: `npm install --ignore-scripts` para validar estáticamente

### 3. Diferencias entre SAE 9 (BD actual) y SAE 10 (versión destino)
- Tablas `INVE01`, `FACTF01`, `PAR_FACTF01` en SAE 9 → pueden llamarse `INVE`, `FACTE`, `FACTT` en SAE 10
- **Validar nombres exactos durante Fase 1** (Discovery) cuando se instale en PC del cliente
- Queries parametrizadas facilitan el ajuste sin reescribir lógica

### 4. Campo `environment` case-sensitive
- **Bug detectado en vivo:** `"QUA"` (mayúsculas) → HTTP 403, `"qua"` (minúsculas) → HTTP 502
- **Solución aplicada:** el config ahora tiene `"qua"` (validado en vivo via UI)
- **Recomendación:** documentar en el manual de operación que siempre sea minúsculas

### 5. Persistencia de historial de ejecuciones
- **Actual:** en memoria (se reinicia con el proceso)
- **Mejora futura:** persistir en SQLite o archivo JSON para auditoría histórica
- No es bloqueante para go-live

### 6. Mapeo de unidades de medida
- SAE guarda `"pz"` (piezas); ejemplo de Siemens usa `"each"`
- **Decisión actual:** mantener `"pz"` (no se transforma)
- **Pendiente:** confirmar con cliente/Siemens cuál es el esperado

### 7. Tarifa horaria preferencial post go-live
- $1,500 MXN/hora para clientes de implementación (vs $1,800 estándar)
- Intervenciones por evento, mínimo 2h, factura única al cierre
- 15 días de garantía incluidos
